Workflow definition

        
{
  "steps": [
    { "id": "fetch", "name": "Fetch data", "type": "http", "config": { "method": "GET", "url": "https://api.example.com/leads" } },
    { "id": "summarize", "name": "Summarize", "type": "ai", "config": { "prompt": "Summarize ${{ steps.fetch.json }}", "system": "Be brief." } },
    { "id": "notify", "name": "Notify Slack", "type": "slack", "config": { "text": "Summary: ${{ steps.summarize.text }}" } }
  ]
}

        
Triggers

        
    
          
  • manual — run from the dashboard or via POST /workflows/{id}/run
    • 
          
  • schedule — cron-like, 5 or 6 fields (e.g. 0 9 * * 1 for Mon 09:00 UTC)
    • 
          
  • webhook — POST /api/v1/public/webhook/{workflow_id}
    • 
        

        
Built-in step types

        
    
          
  • http — call any HTTP endpoint (GET/POST/PUT/DELETE/PATCH)
    • 
          
  • ai — call OpenAI / Anthropic / stub LLM
    • 
          
  • email — send SMTP email (or preview in dev)
    • 
          
  • slack — post a message via Slack incoming webhook
    • 
          
  • transform — extract a JSON value with JMESPath
    • 
          
  • condition — compare and branch
    • 
          
  • delay — pause for up to 30 seconds
    • 
          
  • log — record a log line
    •
